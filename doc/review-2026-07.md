# Calcumaker firmware — code & architecture review (2026-07)

Method: three independent review passes (correctness bug-hunt, architecture/
organization, embedded-readiness) over `firmware/` at ~196 tests, each
verifying findings by running the REPL/emulator, plus the author's own
analysis. Numeric findings were cross-checked against Python.

**Status legend:** ✅ fixed in the review-fix commit · ⏳ deferred (tracked
below, in priority order).

## Verdict in one paragraph

Healthy codebase wearing one big file. The layering (engine → App → seg7 →
thin frontends), the validate-before-pop error discipline, the exactness
contract, and the generated-plus-golden-tested keymap docs were each singled
out as exemplary. The real weaknesses clustered in three seams — the
word-mode canonical-range invariant wasn't enforced at five side-channel
producers, the FIX display path re-derived rounding by string surgery, and
precision was captured at object birth — plus one architecture symptom (the
entry buffer round-tripping through the string command matcher). All were
localized; all P0/P1 items are now fixed.

## Correctness findings

| Sev | Finding | Status |
|-----|---------|--------|
| HIGH | Hex entries spelling command names were stolen by the command table: `E` pushed Euler's e, `DEC` switched radix, `0xCF` was unenterable (App::flush → `input()` matches commands first) | ✅ new `Calc::push_number` number-only door; App entry uses it |
| HIGH | Word-mode canonical-range invariant violated by `rcl`/`lastx`/`ddays`/`dow`/`sigma` → reachable **panics** in `lj` (bit_len underflow) and `popcnt` (negative "pattern") | ✅ producers canonicalize via `push_int_canon`; lj/popcnt mask defensively |
| HIGH | FIX truncated instead of rounding across a decade: `0 fix 999.6` → `999` (exponent probe overshoot for mantissas in [9.95, 10)) | ✅ re-split at the true exponent's digit count |
| MED | `Integer::from_i64` truncates through `c_long` (i32 on arm-none-eabi) → `ran` broken on the MCU while all host tests pass | ✅ portable 31-bit-half assembly + full-range regression test |
| MED | `prec` unbounded (`4e9 prec` → next real op allocates ~500 MB → device OOM); `fact` uncapped; unbounded `shl`/`bset`/`maskr` counts uncapped | ✅ MAX_PREC 16384; n·log₂n guard on fact; count guards ≤ MAX_POW_BITS |
| MED | Σ/TVM/cash-flow registers froze at creation-time precision (Float ops take the LEFT operand's precision) — stale digits displayed as significant | ✅ `set_prec` re-rounds all live registers |
| MED | Classic4: Σ+/Σ− left stack lift enabled (15C disables) → the n in X lifted into Σy, silently poisoning ȳ/s/L.R. | ✅ lift-disable list extended |
| LOW | Word-mode `÷` didn't set carry on an inexact quotient (16C does) | ✅ |
| LOW | `rl`/`rr` silently reduced counts mod n; 16C errors above the word size | ✅ count > n errors (count = n allowed as identity — documented deviation) |
| LOW | `corr`/`lr` NaN on degenerate data | ✅ (fixed during the SCI batch) |
| LOW | Multi-root IRR returns the first bracketed root | documented behavior (12C caveat) |

Confirmed sound under adversarial probing: errors-never-consume (everywhere,
incl. classic4 at exactly 4 levels for every op), encode/decode/wrap down to
1-bit words in all three sign modes, date round-trips at both calendar
boundaries, bisect_rate under stress, LASTx semantics.

## Structural recommendations (⏳ deferred backlog, priority order)

1. **Split `calc.rs` (~2.7k lines) into `calc/{mod,word,ops,trig,stats,finance,dates}.rs`** —
   the banner comments already partition it; mechanical `impl Calc` moves.
   Split `tests/engine.rs` the same way (directory module, ONE test binary).
   (M effort, high payoff, low risk.)
2. **`Modes` sub-struct + snapshot/restore API** — 21 fields on `Calc`; ten
   are display/interaction modes. Grouping them decouples `format.rs`
   (reads only modes) and is the enabler for **continuous memory**, which is
   currently blocked: no constructors/getters for stack/regs/stats/tvm/cfs/
   rng, and `gmp-mpfr-nostd` lacks `mpz_export/import` +
   `mpfr_get_z_2exp/set_z_2exp` for exact binary serialization (do NOT
   round-trip decimal strings). Design: per-value precision, presence
   bitmaps for the Option-al substructs, version byte + CRC, A/B flash
   pages (8 KB erase on U575), ~2.5 KB per snapshot.
3. **Cargo workspace at `firmware/`** with `exclude = ["calcumaker-fw"]` —
   one lockfile/target dir for the host crates; fw keeps its pinned target.
   Fold or delete the empty `common/`/`shared/` placeholders.
4. **`App::press_key` mode enum** — replace the setup/status/pending-reg
   boolean ladder with `enum Mode { Idle, Setup(usize), Status,
   RegSelect(RegOp) }`; quirk found: STO-pending survives a SETUP detour.
   Fold the TVM store/solve table into `token_for(key, solving)`.
5. **De-duplicate label/cycle logic** — `SignMode::next()`/`AngleMode::next()`
   + `label()` methods consumed by calc commands, SETUP, STATUS, and the
   emulator (currently 3–4 hand-synced copies). Strengthen
   `all_personality_keys_dispatch` to also reject `"parse error"` (catches
   token typos, closing the last dispatch-drift hole).
6. **Error display story for the glass** — none of the ~50 error strings are
   7-seg renderable. Decide: HP-style numbered errors (`Error 4`) with long
   strings kept for host frontends, or a `CalcError::glass()` short form.
   Also make `Parse(String)` allocation-free (the App discards the payload).
7. **Keys with no keymap slot** — `abs` and `e` (the constant) have no key in
   ANY personality; audit the 37 token-only functions for oversights.
   `Key::Shl` maps to token `sl` while token `shl` is shift-by-X — document
   the token grammar in one table.

## Embedded-readiness (⏳ for firmware bring-up)

- **Heap vs guards**: 64 KB heap is plausible for the routine workload (TVM
  solve ≈ 1 KB live; IRR-20 fine; 100! trivial) but a 1 Mbit `pow` needs
  ~800 KB transient to *display*. Either grow the heap (256–384 KB of the
  786 KB SRAM) or shrink MAX_POW_BITS on the device build. ✅ prec/fact/count
  guards landed in the fix pass; heap sizing is a fw decision.
- **Never-hard-fault bounds**: cap the unbounded stack depth (e.g. 256),
  cash-flow groups (99), and entry length (~256 chars) in firmware config.
- **Panic story**: panic-halt is unacceptable — write a crash marker to
  SRAM4/backup domain, `sys_reset()`, show `Error 9` + restore the snapshot
  on boot. Provide `abort` for GMP (it calls abort on alloc failure — GMP
  cannot unwind), set MSPLIM, add IWDG. GMP builds with alloca temporaries
  on the C stack — either budget stack or `--enable-alloca=malloc-reentrant`.
- **Display latency**: `text_rows` formats the WHOLE stack (and `window()`
  re-formats) every keypress — format only the visible rows + cache X's
  string; a 300k-digit integer anywhere on the stack currently costs seconds
  per refresh.
- **Abort hook**: thread a `keep_going: &mut dyn FnMut() -> bool` through
  `bisect_rate` and the DB-depreciation loop (the only long loops) → new
  `CalcError::Aborted`; firmware polls the key matrix. Solve latency is
  otherwise period-authentic (50 ms–2 s @256-bit); adaptive convergence
  (stop at display precision, Newton-polish) would cut it 5–10×.
- **Missing firmware APIs**: `Calc::set_rng_seed(u64)` (hardware entropy),
  an `AppEvent` return from `press_key` for `Key::Off` (power-down signal).
- **Misc**: bind + call `mpfr_free_cache` on prec changes (π cache);
  `mp_set_memory_functions` must be wired before any Integer/Float exists;
  `keydoc` will be stripped by LTO (verify with `nm --size-sort` after first
  target link); the lone `f64` in `dec_digits` links soft-double fine.

## What the reviewers called out as good

Single numeric path with no fallback; validate-before-pop applied everywhere
and tested as a contract; exact-integer promotion rules documented at each
site; device-independent App/seg7 with byte-identical emulator/firmware
display; generated keymap docs guarded by golden tests; behavior-level test
suite (200+) that makes every refactor above low-risk.
